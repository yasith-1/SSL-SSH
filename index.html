SSL / TLS කියන්නේ මොකක්ද?

Web browser එකක් සහ Web server එකක් අතර data ආරක්ෂිතව යැවීම

නිර්වචනය

SSL (Secure Sockets Layer) — දැන් ගොඩක් වෙලාවට TLS (Transport Layer Security) ලෙස හඳුන්වනු ලබයි. මේක HTTP connection එකට security layer එකක් add කරලා HTTPS බවට පත් කරනවා. Browser address bar එකේ 🔒 padlock icon එක දකිනවා නේද? ඒක SSL/TLS ක්‍රියාත්මක බවේ සලකුණ.

SSL ඇතිව

Details ටික යකඩ පෙට්ටියක දාලා යවනවා වගේ. යතුර තියෙන්නේ server ගාව විතරක් — කවුරු ගත්තත් ඇරලා බලන්න බෑ.

තවත් වැදගත් කරුණු

Let's Encrypt (Free SSL): ඉස්සර SSL Certificates මුදල් දී ලබාගැනීමට සිදු වුවත්, දැන් ලෝකයේ බොහොමයක් websites වලට (Spring Boot / Next.js backend වලටත්) නොමිලේම SSL ලබා දෙන්නේ Let's Encrypt හරහාය.

SSL Handshake: Browser එකයි Server එකයි data යවන්න කලින් "අපි කොහොමද මේක encrypt කරන්නේ?" කියලා කතා කරගන්න මූලික ක්‍රියාවලියට SSL Handshake එක යැයි කියනු ලැබේ.

SSH

SSH කියන්නේ මොකක්ද?

Remote server එකකට internet හරහා ආරක්ෂිතව connect වී control කිරීම

නිර්වචනය

SSH (Secure Shell) — එක computer එකක ඉඳන් තවත් computer එකක (ගොඩක් වෙලාවට remote server) terminal access ලබා ගන්න use කරන protocol එකක්. ඔයාගේ keyboard commands encrypted tunnel එකක් හරහා server එකේ run වෙනවා. AWS, DigitalOcean, GitHub — සෑම තැනකම SSH use කරනවා.

🌐

SSH නැතිව

Server password එක plain text විදිහට internet එක හරහා යනවා. Hacker ලාට ලේසියෙන් steal කරලා server hijack කරන්න පුළුවන්.

🚇

SSH ඇතිව

රහස් underground tunnel. ඔයා type කරන commands කෙලින්ම ගිහින් server එකේ run වෙනවා. පිටින් ඉන්න කිසිම කෙනෙකුට ඒක පේන්නේ නෑ.

තවත් වැදගත් කරුණු

SSH Port Forwarding (Tunneling): Remote Server එකක පිටින් access කරන්න බැරි Database එකකට (උදා: MySQL port 3306) අපේ local මැෂින් එකේ ඉඳන් ආරක්ෂිතව connect වෙන්න SSH Tunneling භාවිතා කළ හැක. මේක developer කෙනෙක්ට තියෙන ලොකුම super-power එකක්.

File Transfer (SCP/SFTP): SSH මගින් terminal commands පමණක් නොව, scp හෝ sftp භාවිතා කර ආරක්ෂිතව server එකට files යැවීමද (උදා: build කළ React/Next.js folder එකක්) සිදු කළ හැක.

TERMINAL — SSH Connect 
# 1. Password method — server connect
$ ssh ubuntu@203.0.113.10
# → password: ************ (encrypted over network)

# ─────────────────────────────────────────────────────
# 2. SSH Key Pair Generate කිරීම (ආරක්ෂිතම ක්‍රමය)
$ ssh-keygen -t rsa -b 4096 -C "dev@mycompany.lk"
# → ~/.ssh/id_rsa      (Private Key — කිසිදා share කරන්නේ නෑ!)
# → ~/.ssh/id_rsa.pub  (Public Key  — server / GitHub ට දෙනවා)

# 3. Key use කර connect
$ ssh -i ~/.ssh/id_rsa ubuntu@203.0.113.10
# → Password typed කරන්නේ නෑ! Key authentication.
TERMINAL — Docker Deploy via SSH 
# Server connect වූ පසු — Deploy workflow
ubuntu@server:~$ docker pull myapp:latest
ubuntu@server:~$ docker stop myapp-container
ubuntu@server:~$ docker run -d \
    --name myapp-container \
    -p 8080:8080 \
    -e SPRING_PROFILES_ACTIVE=prod \
    myapp:latest

ubuntu@server:~$ sudo systemctl restart nginx
# ✅ Deploy done — SSH tunnel හරහා safely
TERMINAL — GitHub SSH Setup 
# 1. Public key copy කරලා GitHub ට add කරන්න
#    GitHub → Settings → SSH and GPG Keys → New SSH Key
$ cat ~/.ssh/id_rsa.pub
# → ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAABgQ... (paste in GitHub)

# 2. Remote URL SSH ට change කිරීම
$ git remote set-url origin git@github.com:username/project.git

# 3. SSH test
$ ssh -T git@github.com
# → Hi username! You've successfully authenticated.

# 4. Push / Pull — now via SSH
$ git push origin main   # Password type කරන්නේ නෑ!

SSL vs SSH — සංසන්දනය

Differences, similarities, සහ use cases side-by-side

ලක්ෂණය 🔒 SSL / TLS 🔑 SSH
Full Name Secure Sockets Layer / Transport Layer Security Secure Shell
ප්‍රධාන අරමුණ Browser ↔ Server data encrypt කිරීම Remote server control & management
Port Number 443 22
Protocol HTTPS (HTTP over TLS) SSH protocol
User Interface Browser padlock 🔒 icon Terminal / Command Line
Use කරන්නේ End-users, Web Apps, APIs Developers, System Admins
Authentication SSL Certificates (CA issued) Password හෝ SSH Key Pair
Real-world Example Bank login, online payment, hotel checkout GitHub push, AWS deploy, Docker run
Analogy යකඩ පෙට්ටියක දාලා data යැවීම 📦 රහස් underground tunnel 🚇
Encryption Asymmetric + Symmetric (TLS handshake) Asymmetric Key Pair (RSA/Ed25519)

  • React / Vue frontend API calls

  • Payment / Checkout endpoints

  • Sinhala: "Data travel කරද්දී ආරක්ෂා"

🔑 SSH — කොහෙදැ Use කළ යුතු?

  • AWS EC2 / DigitalOcean VPS login
  • GitHub / GitLab code push & pull
  • Docker image deploy & manage
  • Nginx / Apache web server config
  • Database backup & maintenance
  • Sinhala: "Server ට connect වෙලා control"
දෙකෙම සමාන දේ

SSL සහ SSH දෙකම encryption use කරනවා — නමුත් අරමුණ සහ context සම්පූර්ණයෙන්ම වෙනස්. SSL කියන්නේ end-user ගේ browser request ආරක්ෂා කිරීම. SSH කියන්නේ developer / admin ගේ server management ආරක්ෂා කිරීම. Real project එකක දෙකම use කරනවා — HTTPS API calls (SSL) + server deploy (SSH).
